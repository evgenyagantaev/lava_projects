# LIF нейрон с анимацией (lava-nc)

Небольшой проект: WebSocket-сервер моделирует одиночный LIF-нейрон, а фронтенд на HTML/CSS/JS рисует анимированную сцену (дендриты, аксон, графики мембранного потенциала и поезд спайков), где данные движутся в реальном времени справа налево.

## Файлы
- `server.py` — лёгкий WebSocket-сервер, который шаг за шагом обновляет LIF и шлёт кадры в реальном времени.
- `static/index.html`, `style.css`, `app.js` — визуализация и анимация (только live-данные из WebSocket).
- `backend.py` — вспомогательный скрипт симуляции (можно использовать для офлайн экспериментов, но фронтенд его не читает).

## Запуск (live-only)
1. Установите зависимости (в активном окружении с lava-nc):
   ```bash
   pip install lava-nc websockets
   ```
2. Запустите live-стрим LIF:
   ```bash
   python projects/04_lif_animation/server.py --host 127.0.0.1 --port 8765 --rate 0.1 --dv 0.04 --spike-fraction 0.4 --delay-ms 80
   ```
3. Поднимите статику для фронтенда:
   ```bash
   python -m http.server 8000 --directory projects/04_lif_animation/static
   ```
4. Откройте `http://localhost:8000` в браузере. По умолчанию фронтенд подключается к `ws://<host>:8765/stream`. Если сервер на другом порту/хосте, ниже `app.js` в `static/index.html` пропишите:
   ```html
   <script>window.LIF_WS = "ws://localhost:8765/stream";</script>
   ```

## Настройки симуляции
- `--rate` — вероятность входного спайка на шаг (случайные моменты времени).
- `--spike-fraction` — амплитуда входного спайка как доля порога.
- `--dv` — утечка мембранного потенциала (экспоненциальное уменьшение).
- `--threshold` — порог для генерации выходного спайка; при пересечении потенциал сбрасывается.
- `--delay-ms` — задержка между кадрами, определяет скорость анимации.

При желании можно расширить фронтенд (добавить несколько нейронов/синапсов, подписывать параметры на экране или показывать распределение входных спайков).
